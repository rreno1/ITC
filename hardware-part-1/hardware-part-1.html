02 Basics

What is Hardware?

Hardware refers to the physical, tangible components of a computer system that you can touch and see, operating under the instruction of software.

✓
Tangible vs. Intangible: Hardware is the physical engine (CPU, RAM, drives), while software is the code that drives it.
✓
Under the Hood: Computers appear to understand English, images, and videos, but physically they only process electrical signals.
✓
Digital Abstraction: We abstract physical signals (voltages) into logical values: 0 (off) and 1 (on).
Physical computer parts: motherboard, CPU, RAM stick, and solid state drive.

Detailed Lecture Notes

Physical computer parts represent the machinery that converts raw electrical currents into computing logic. Unlike the software that controls it, hardware is tangible and consists of materials like copper, silicon, and fiberglass. Every transaction in a system must eventually be executed by these physical structures under command loops. Without hardware to run it, code remains a set of instructions with no engine to carry them out. Therefore, a complete understanding of computing requires analyzing how these components function at the physical level.

At the very core of digital computing, everything is reduced to electrical voltage states. Engineers designed microscopic circuits that can represent two distinct options: voltage present or voltage absent. This binary abstraction prevents random voltage variations from corrupting critical calculations within the system pathways. These physical switches are known as transistors, and they act as the base gatekeepers of digital memory. By combining billions of these switches on a single chip, complex calculations can be processed in fractions of a second.

The motherboard serves as the communication central routing board where all physical components meet. Memory modules store active data, while storage drives preserve files permanently even after power loss. Peripherals plug into outer ports to feed new signals into the processor registers. Each component plays a distinct role in balancing the overall throughput speed of the system. Understanding this mechanical division of labor helps developers optimize their software for real hardware targets.

For example, think of a computer as a physical piano, where the keys, hammers, and strings represent the hardware. The sheet music containing musical notes represents the software instructions guiding the performance. Just as a piano cannot produce sound without a pianist pressing physical keys, software cannot compute without hardware switches. Another simple analogy is a physical kitchen where the pans, stoves, and utensils are the hardware tools. The recipe book represents the software instructing the cook on how to combine ingredients using those tools.

03 Logic

The Binary System

Computers use the Binary system (Base 2), which uses only two digits: 0 and 1, differing from the Decimal system (Base 10) we use daily.

1
Transistors: Microscopic switches inside chips. If electricity flows, it represents 1. If blocked, it represents 0.
2
Reliability: It is electronically much easier and more reliable to distinguish between two states (on/off) than ten states (different voltage levels).
3
Base 2 Counting: Every place value in binary represents a power of 2 (1, 2, 4, 8, 16, 32, 64, 128...).
The binary system utilizes base 2 logic (ones and zeros).

Detailed Lecture Notes

The binary system represents the native language of computers, utilizing a base of two numbers. While humans count using ten digits, computers use only zero and one. This mathematical system corresponds directly to the physical states of transistors inside computer microchips. An open circuit allowing current represents a one, while a closed circuit blocking it represents a zero. This clean mechanical mapping makes digital calculations highly reliable and resistant to interference.

Positional notation in binary works similarly to decimal, but each column increases by powers of two. The rightmost column represents ones, the next represents twos, followed by fours, eights, and so on. By adding these place values together, any decimal integer can be represented in binary format. For example, the binary string 101 represents one four and one one, summing to five. This simple base mathematical structure allows computers to perform complex arithmetic using only basic logic gates.

Boolean logic drives all CPU decisions by processing binary inputs to produce a single binary output. Gates like AND, OR, and NOT combine to form complex execution blocks. Millions of these logic gates are etched onto silicon dies to calculate software instructions. This hardware-level math operates at clock speeds measured in billions of cycles per second. Thus, the binary system acts as the bridge connecting physical electricity to mathematical logic.

An easy analogy for the binary system is a long row of ordinary household light switches. If you turn a switch up, it lights up representing a binary one. If you flip it down, the bulb goes dark representing a binary zero. For instance, sending a message across early telegraph wires used short and long pulses to transmit words. By combining these simple light switch states, you can create complex code coordinates to display images.

04 Units

Bits and Bytes

All computer data is built from these fundamental building blocks of digital storage.

✦
Bit (Binary Digit): The smallest unit of data. Can be a 0 or a 1.
✦
Byte: A group of 8 bits. It is the standard base unit of computer storage.
✦
Capacity: A single byte can represent 28 = 256 unique values (ranges from 0 to 255).
✦
Standard Units: Kilobyte (KB ~ 1,000 bytes), Megabyte (MB ~ 1 million bytes), Gigabyte (GB ~ 1 billion bytes), Terabyte (TB ~ 1 trillion bytes).
Bits are individual binary digits grouped together into bytes on storage chips.

Detailed Lecture Notes

The bit is the fundamental building block of all digital information in computer memory. Short for binary digit, a single bit can only hold a value of zero or one. On their own, individual bits cannot store complex files like pictures or audio. However, by grouping bits together, we can represent larger numbers and characters. This grouping of binary units forms the foundation of modern data storage and transmission.

A byte is a group of exactly eight bits and represents the standard storage unit. With eight binary positions, a single byte can represent two hundred and fifty-six unique states. This capacity is sufficient to encode a single letter, number, or punctuation mark. When you view file sizes, they are measured in multiples of these eight-bit bytes. Kilobytes, megabytes, and gigabytes represent thousands, millions, and billions of these storage blocks.

Computer memory addresses are assigned to individual bytes rather than single bits. This allows the CPU to fetch and write data blocks efficiently during execution loops. If a program needs to store a large number, it utilizes multiple adjacent bytes. This byte-addressable architecture simplifies how memory is managed by the operating system. Knowing how bytes group is essential for managing computer storage capacities and networking budgets.

Think of a bit as a single alphabet letter, which has very limited meaning on its own. A byte is like a complete eight-letter word that conveys a specific, readable instruction to the reader. For example, a single character on this screen requires exactly one byte of storage memory space. Another simple analogy is a egg carton that holds exactly eight eggs in designated slots. You can represent different numbers by leaving specific egg slots empty or filled.

06 Encoding

Representing Text

How do computers translate binary numbers into text? We need a mapping standard where each character (letter, number, symbol) is assigned a numeric value.

A
Character Encoding: A lookup table that links binary codes to human-readable characters.
B
Standardization: Without an agreed standard, sending text from one computer to another would result in unreadable gibberish.
C
Major Standards: Over time, two primary systems have dominated computer history: ASCII and Unicode.
Character encoding maps physical key cap letters and symbols to numeric values.

Detailed Lecture Notes

To represent human language, computers need a consistent system to map numbers to characters. Since hardware only processes binary digits, characters must be encoded as numeric values. A character encoding table acts as a dictionary connecting numbers to specific letters. When you type on a keyboard, a numeric code is sent to the processor. The computer then looks up this numeric code to display the correct symbol on screen.

Without a standardized lookup dictionary, files sent between computers would display as unreadable garbled text. Early computer developers created custom tables, which often caused severe document compatibility problems. These localized encoding systems made international communication difficult and prone to corruption. Therefore, standardizing character maps became a priority for the emerging digital network. Modern systems rely on global standards to guarantee that text renders correctly worldwide.

Character encoding is not limited to letters but also covers numbers and control symbols. Control characters include instructions like carriage returns, tabs, and line breaks. The operating system processes these hidden codes to format text layouts properly on screen. This ensures that formatting remains consistent across different software applications and web browsers. Managing these translation tables is the foundation of global software development and data transmission.

A simple analogy for character encoding is the classic Morse code used by early telegraph operators. In Morse code, unique combinations of dots and dashes map to specific letters of the alphabet. For instance, three short taps represent the letter S, and three long taps represent O. Similarly, when you press a key on your laptop, the keyboard sends a numbers-only signal. The system translates this signal using a lookup map to display the character on screen.

07 ASCII

ASCII Encoding

ASCII (American Standard Code for Information Interchange) was defined in 1963 for telegraph systems and later adapted for computers.

1
Size: Originally a 7-bit code, representing 27 = 128 characters.
2
Range: Includes English uppercase/lowercase letters, numbers 0-9, basic punctuation, and control characters (like newline).
3
Examples: Letter 'A' is mapped to 65 (binary: 01000001). Letter 'a' is mapped to 97 (binary: 01100001).
ASCII defines a 7-bit standard mapping for English characters on early computer systems.

Detailed Lecture Notes

ASCII stands for the American Standard Code for Information Interchange and was defined in 1963. It was originally designed for telegraph communication but became the standard for early computer systems. ASCII utilizes a seven-bit encoding structure to map characters to numbers. This seven-bit length allows it to represent exactly one hundred and twenty-eight unique characters. These values include the standard English alphabet, digits, punctuation marks, and control codes.

In the ASCII table, uppercase letters are grouped sequentially to make sorting operations easy. For example, uppercase A is mapped to sixty-five, while uppercase B is sixty-six. Lowercase letters follow a similar sequence, starting at decimal ninety-seven with lowercase a. This numeric arrangement allows programmers to manipulate text strings using simple arithmetic. For instance, subtracting thirty-two from a lowercase letter code converts it to uppercase.

Although ASCII is very simple, its small file footprint made it highly efficient for early networks. Because it only required seven bits, it fit comfortably inside standard eight-bit bytes with room to spare. The remaining eighth bit was often used as a parity check to detect transmission errors. This efficiency helped early computers maximize their limited storage and communication bandwidth. Today, ASCII remains the historical baseline upon which modern encodings are built.

Think of ASCII as a simple pocket dictionary containing only one hundred and twenty-eight English words. If you try to translate a foreign language using this dictionary, you will find no matching entries. For example, typing the letter H corresponds to decimal seventy-two in the ASCII index. If you want to type the symbol for a euro currency, ASCII has no code for it. This makes it like a post office that only delivers letters to English-speaking regions.

08 Limitations

The Limitations of ASCII

As computing went global, ASCII's 128-character limit quickly became insufficient for international communication.

✗
English-Centric: ASCII has no characters for accents (e.g., é, ü), which are common in European languages.
✗
Foreign Scripts: It cannot represent writing systems like Chinese, Japanese, Arabic, Cyrillic, or Greek, which require thousands of characters.
✗
Emojis: Popular modern communication elements (e.g., 😂, 🚀, 💻) have no place in ASCII's simple 7-bit table.
ASCII cannot support accents, international scripts, or emojis on modern devices.

Detailed Lecture Notes

The primary limitation of ASCII is its tiny seven-bit capacity of one hundred and twenty-eight characters. This small range can only fit the basic English alphabet and common symbols. As computing expanded globally, this English-centric design became a major obstacle. Languages that use accented letters, such as Spanish, French, and German, could not be mapped. This limitation meant ASCII was unable to support the writing systems of most global populations.

Non-Latin scripts like Chinese, Arabic, Cyrillic, and Hindi require thousands of unique characters. Trying to fit these complex writing systems into a 128-character table is mathematically impossible. To solve this, developers created localized eight-bit extensions that mapped characters to the extra values. However, these custom tables clashed with each other and caused documents to display incorrect symbols. A document written in one country would open as scrambled gibberish in another.

Modern text elements, like emojis, also require representation in digital files. Because ASCII was created long before emojis existed, it has no codes allocated for them. Without a larger encoding standard, modern digital expression would be severely limited. These combined shortfalls made it clear that a new universal standard was necessary. Moving beyond ASCII allowed computing to become a truly global, multilingual medium.

A good analogy is trying to write a global newsletter using only a standard typewriter. If the typewriter keys lack accents or non-English letters, you cannot print foreign words. For instance, writing the word Café requires an accented letter that standard ASCII cannot display. If a Russian or Chinese document opens in an ASCII editor, it renders as corrupt text blocks. This is like a postman returning a letter because the destination address is written in characters he cannot read.

09 Unicode

Unicode: The Universal Standard

Created to unify all encoding systems, Unicode aims to represent every character in every human writing system, past and present.

✓
Huge Capacity: Capable of representing over 1.1 million symbols (currently defines ~149,000).
✓
UTF-8: The most dominant encoding on the web. It is variable-width (1 to 4 bytes).
✓
Backward Compatibility: The first 128 characters of UTF-8 are 100% identical to ASCII. A standard ASCII file is also a valid UTF-8 file!
Unicode establishes a precise, universally standardized measurement mapping for global text.

Detailed Lecture Notes

Unicode was created to establish a single, universal character set for all written human communication. Its goal is to represent every character from every language, past and present. Currently, Unicode has defined over one hundred and forty-nine thousand unique characters. It contains symbols for historical scripts, mathematical equations, and standard global emojis. This massive directory ensures that no two characters share the same numeric identifier.

UTF-8 is the most popular encoding standard used to implement Unicode on the web. It is a variable-width encoding, meaning characters can take between one and four bytes. This flexible format optimizes storage because standard English text still only uses one byte. Rare characters and emojis scale up automatically to utilize more bytes as needed. This variable design allows UTF-8 to balance storage efficiency with universal language compatibility.

A key feature of UTF-8 is its complete backward compatibility with standard ASCII files. The first one hundred and twenty-eight codes in UTF-8 are identical to the ASCII table. This means any legacy ASCII text file is already a valid UTF-8 file. This smart design prevented the corruption of old archives when the web transitioned to Unicode. Today, UTF-8 is the default standard for almost all digital text transmission.

Think of Unicode as a massive global dictionary containing every word from every language on earth. It acts like a giant catalog where every unique character gets a permanent, universal item number. For example, the thumbs-up emoji has a specific Unicode identifier that displays identically across all devices. UTF-8 functions like a smart shipping box that changes size depending on what is placed inside. Small letters get a small box, while complex characters and emojis get a larger box.

12 Conclusion

Hardware Part 1 Summary

  • Hardware represents the physical machinery; all logic is built on electrical transistors.
  • Binary is the Base-2 system used by hardware (composed of bits and bytes).
  • 1 Byte = 8 Bits, providing 256 unique combinations.
  • ASCII was the early 128-character standard, which was English-only.
  • Unicode (UTF-8) is the modern global standard, supporting all languages, symbols, and emojis.
  • Next Week: We will explore the core internal components inside a computer in Hardware Part 2 (Motherboards, CPUs, RAM, and Storage).
Review of fundamental topics inside the computer unit.

Detailed Lecture Notes

Hardware represents the physical foundation of all modern digital computer systems. All logic inside a machine is managed by manipulating electrical voltage levels. We abstract these physical voltages into binary values of zeros and ones. These binary values form the native language of the processor chips. By combining these binary states, computers can represent complex data structures.

Data capacity scales up from individual bits to groups of eight-bit bytes. Bytes are the standard units of computer memory allocation and file storage. To represent text, computers map these byte values using character tables. ASCII was the early standard, but it was limited to basic English. Unicode (UTF-8) resolved this limit by mapping all global languages and emojis.

Programmers must understand how physical hardware abstracts data to write efficient software. Mismatched character encodings can cause data loss and corrupt database entries. Modern web standards require using UTF-8 to ensure universal communication compatibility. In the next module, we will explore the core internal components of computer systems. We will look at motherboards, processor execution loops, RAM memory, and storage drives.

A final analogy is to view a computer system as a modern busy post office. The physical trucks, mailboxes, and sorting tables represent the hardware components. The mail packets flowing through the system are the bits and bytes of raw binary data. The address formats written on the envelopes correspond to character encoding standards like Unicode. Understanding how these layers connect helps you see the physical logic behind modern digital networks.
