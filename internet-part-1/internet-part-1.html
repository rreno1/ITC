02 Nodes

Internet Fundamentals & Network Devices

Computers on a network occupy specific functional roles to request, distribute, and coordinate raw data packets.

✓
Clients: The devices we use to request files (smartphones, laptops running web browsers).
✓
Servers: Powerful remote computers that store web files, processing requests and returning answers.
✓
Routers: Traffic coordinators. They check target addresses and direct packets along the fastest pathways.
Network client connections require grounded connection links.

Detailed Lecture Notes

The internet operates as a distributed network of client and server machines. Clients represent the endpoint devices, like laptops or smartphones, that request web files. Servers are powerful remote host systems that store databases and serve requests. When you open a browser, it issues a request to a server node. The server coordinates the response by sending back files containing layout data.

Connecting these endpoints together is a complex grid of routing devices. Routers act as traffic coordinators that guide packets to their destination addresses. They inspect the destination header of each incoming packet of information. Using path routing tables, they direct the data along the fastest path. Without these network routers, data packets would get lost in transit.

The client-server relationship is structured around standard software request cycles. A user initiates a transaction by clicking a web link icon. The request travels through local routers out to the wider web. The host server receives the transaction details and replies with information. Understanding this structural division is essential for troubleshooting web connectivity errors.

A simple analogy is ordering a package from a large online retail warehouse. You are the client requesting an item using your web browser catalog. The retail warehouse is the server hosting the items and dispatching deliveries. The mail carrier delivery trucks acting as path finders represent network routers. They inspect the address labels to drive packets along optimal highway routes.

03 Addressing

MAC, IPv4, IPv6 & Network Prefixes

An IP (Internet Protocol) Address is a unique numerical label assigned to every device on a network so packets know where to go.

1
IPv4 Standard: 32-bit addresses written in dotted-decimal format (e.g. 192.168.1.1).
2
Four Octets: Four numbers separated by dots. Each number is 8 bits (range from 0 to 255).
3
Network vs Host: The IP designates both the local network path and the specific device interface (host node).
Octet formatting divides address fields for precise packet delivery routing.

Detailed Lecture Notes

Every device connected to the internet must have a unique identifier. This identifier is called an IP address, representing the Internet Protocol. IP addresses act like mailing addresses, ensuring information reaches correct hosts. The most common standard is IPv4, composed of thirty-two bits. These addresses are formatted as four numeric octets separated by dots.

An octet represents an eight-bit block holding a range from 0-255. An example of an IPv4 address is one hundred ninety-two dot one. This format is easily read by humans but processed as binary. Each IP address is divided into network and host identification portions. Subnet masks specify where the network section ends and hosts begin.

If two computers share the exact same IP, address conflicts occur. This collision causes both hosts to lose connection to the network. Standard public addresses are registered and assigned by global naming authorities. Local network routers manage private namespaces to prevent these conflicts. Understanding IP formatting is the baseline of modern network configuration steps.

Think of an IP address as a standard telephone number with area codes. The area code designates the city network, while local digits identify the phone. For example, the network portion of the IP groups devices in the same office. The host portion identifies your specific laptop connected to that router. Without these unique numbers, you could not dial and connect to other hosts.

04 Exhaustion

Packet Switching & the TCP/IP Model

IPv4 capacity is limited to 232 ~ **4.3 billion** addresses, which has been fully exhausted by global smartphone and IoT expansion.

1
IPv6 Standard: A 128-bit address architecture providing 2128 (undecillions) of unique address nodes.
2
Hexadecimal Format: Grouped into eight blocks separated by colons (e.g. 2001:db8::ff00:42:8329).
3
Hex Digits: Uses numbers 0-9 and letters A-F. Eliminates the need for Network Address Translation.
Physical network links carry packets between connected equipment. Photo: ProjectManhattan, CC BY-SA 3.0, via Wikimedia Commons.

Detailed Lecture Notes

IPv4 provides a theoretical maximum of four point three billion addresses. When the internet was designed, this capacity seemed more than sufficient. However, the explosion of smart devices led to IPv4 address exhaustion. Every smartphone, laptop, and smart appliance required its own unique link. This rapid growth forced the industry to develop a replacement standard.

The new standard is IPv6, which uses a 128-bit architecture. This massive increase provides octillions of unique address combination paths. Instead of decimal numbers, IPv6 is written in hexadecimal notation. Hexadecimal uses sixteen characters: digits zero through nine and letters A-F. This design ensures the world will not run out of addresses.

Transitioning the global internet to IPv6 is a long, gradual process. Many routers must run dual-stack configurations supporting both protocols concurrently. IPv6 also offers built-in security features and automated configuration routines. It eliminates the need for complex address translations on local routers. Studying IPv6 formats is critical for modern network engineers and administrators.

A good analogy is a rapidly expanding city running out of telephone numbers. To solve this, the city adds extra digits to every phone prefix number. For instance, IPv4 was like having a seven-digit phone system that filled up. IPv6 acts like a twenty-digit numbering system that will never run out of lines. This massive directory ensures every smart sensor on earth gets a unique identifier.

05 Protocols

Automatic Configuration: DHCP, NAT & Gateways

How devices within local spaces connect securely without wasting globally registered addresses.

✓
Private IPs: Addresses like 192.168.1.x. Used internally. Routers translate these to a single public IP to route onto the web.
✓
DHCP Role: Dynamic Host Configuration Protocol. Automates address distribution by leasing IPs to client devices.
✓
DORA Cycle: The four-step handshaking exchange: **Discover** (client), **Offer** (server), **Request** (client), **Acknowledge** (server).
DHCP lease expiration disconnects old clients to recycle inactive address pools.

Detailed Lecture Notes

IP addresses are divided into public and private scopes for routing. Public IP addresses are globally unique and accessible across the web. Private IP addresses are reserved for internal home or office networks. Local routers use Network Address Translation to link private addresses out. This division conserves public IPv4 addresses and enhances local system security.

DHCP stands for Dynamic Host Configuration Protocol, an automated system. Manually assigning IP addresses to every connected device is highly inefficient. Instead, a DHCP server leases temporary IP configurations to client machines. When you connect to Wi-Fi, your device automatically requests an IP. The router assigns a lease from a pre-configured local pool.

The DHCP lease process consists of four basic transaction exchange stages. These stages are Discover, Offer, Request, and Acknowledgment, forming DORA. A client broadcasts a discover message looking for available network leaseholders. The server responds with an offer containing a valid local address. Once acknowledged, the client binds the configuration until the lease expires.

Think of a private IP as an office extension number like extension four-one. You can call extensions inside the building, but outsiders cannot dial them directly. The main company telephone number represents the public IP routing traffic. DHCP acts like a hotel receptionist assigning room keys to checking-in guests. Once a guest checks out, the key IP is recycled for the next visitor.

06 Physical Layer

Physical Internet Infrastructure

All logical binary packets must ultimately travel over physical mediums acting as electrical, optical, or wireless transceivers.

✦
Copper Cables (Ethernet): Uses electrical voltage pulses down copper twisted-pairs. Reliable, cheap, but limited to short distances.
✦
Fiber-Optic Cables: Transmits binary as light flashes inside glass strands. Extreme speeds over massive distances.
✦
Wireless Waves: Transmits using radio microwave bands. Wi-Fi links local scopes, while cellular 4G/5G links wider areas.
Network routing ports manage physical links via optical fiber transceivers.

Detailed Lecture Notes

The physical layer represents the actual medium carrying data signals. All binary information must be converted into physical pulses for travel. These pulses can be electrical currents, light beams, or microwaves. The choice of physical medium determines the speed and range limit. Every network connection depends on these base transmission components.

Copper Ethernet cables transmit binary digits using varying electrical voltage states. They are cheap and reliable but limited to short distances. Fiber-optic cables transmit information as rapid pulses of laser light rays. These light signals travel down thin glass strands at incredible speeds. Fiber-optics form the high-speed backbone connecting continents across undersea trenches.

Wireless transmissions rely on radio frequency waves to send binary blocks. Cellular networks use tower arrays to transmit data over wide areas. Modern 5G links deliver massive bandwidth but have limited obstacle penetration. Each physical medium has distinct advantages in cost, speed, and deployment. Understanding these physical cables is essential for troubleshooting connection speed drops.

A simple analogy is sending messages using different physical transit channels. Copper Ethernet cables are like sending notes down metal tubes using electricity. Fiber-optic cables are like sending Morse code flashes down long glass pipes. Wireless Wi-Fi is like shouting words across a room using air wave vibrations. Each medium converts binary zeros and ones into physical pulses for travel.

10 Conclusion

Internet Part 1 Summary

  • The **Internet** is distributed infrastructure connecting networks; the **Web** is one service using it.
  • NICs, switches, access points, routers, modems/ONTs, firewalls, ISPs, IXPs, and servers each have distinct roles.
  • Data is divided into packets; the TCP/IP layers encapsulate and deliver it across possible routes.
  • MAC, IPv4/IPv6, public/private addresses, prefixes, gateways, DHCP, and NAT solve different addressing tasks.
  • Ports identify services; routing tables, next hops, TTL, autonomous systems, and BGP enable distributed routing.
  • TCP and UDP serve different application needs; bandwidth, throughput, latency, jitter, loss, and congestion describe performance.
Review of connectivity nodes, IP directories, and physical routing layers.

Detailed Lecture Notes

Internet Part 1 established the core foundations of network data transit. We explored the client-server relationship and the routing of packets. Clients request web files, while servers store and distribute them. Routers guide these packets along path channels using target IPs. Every device needs a unique address to participate in this network.

IPv4 is the thirty-two bit standard separated into four numeric octets. IPv6 is the modern hexadecimal standard utilizing a 128-bit address. Public IP addresses route globally, while private addresses manage local spaces. DHCP automates configuration by dynamically leasing local IPs using DORA. The physical layer carries these binary pulses over copper, fiber, or air.

The practical TCP/IP model has four layers: application, transport, internet, and network access. TCP manages ordered reliable delivery; UDP provides connectionless datagrams with less overhead. Neither is universally “fast” or “slow”: applications choose according to reliability, timing, and latency needs.

A final analogy is to compare network infrastructure to a global shipping port. The shipping containers represent binary packets traveling across physical layers. The unique port registration codes correspond to IPv4 and IPv6 addresses. The automated dock managers leasing storage bays function like DHCP servers. Together, these layers connect local interfaces to the wider global internet.
