02 Transit

Packets & Routing

Files sent over the internet are not transmitted as single, massive chunks. They are sliced into smaller units called **packets** for efficient routing.

✓
Data Slicing: Files (images, HTML pages) are broken into packets, typically limited to 1,500 bytes.
✓
Header Metadata: Each packet contains header tags listing the source IP, destination IP, and sequence numbers.
✓
Independent Routing: Packets travel across different routing hops independently, reassembling at the destination.
Large files are sliced into smaller packet blocks for web routing.

Detailed Lecture Notes

The internet operates as a distributed network of client and server machines. Clients represent the endpoint devices, like laptops or smartphones, that request web files. Servers are powerful remote host systems that store databases and serve requests. When you open a browser, it issues a request to a server node. The server coordinates the response by sending back files containing layout data.

Connecting these endpoints together is a complex grid of routing devices. Routers act as traffic coordinators that guide packets to their destination addresses. They inspect the destination header of each incoming packet of information. Using path routing tables, they direct the data along the fastest path. Without these network routers, data packets would get lost in transit.

The client-server relationship is structured around standard software request cycles. A user initiates a transaction by clicking a web link icon. The request travels through local routers out to the wider web. The host server receives the transaction details and replies with information. Understanding this structural division is essential for troubleshooting web connectivity errors.

A great analogy is mailing a large printed textbook to a friend overseas. Instead of mailing the heavy book, you rip out every page separately. You put each page in its own envelope marked with a sequence number. The postal system routes the envelopes along different flights and paths. Once they arrive, your friend reorders the pages using the sequence numbers.

03 TCP/IP

TCP/IP: Handshake & Delivery

While the **Internet Protocol (IP)** handles addressing, the **Transmission Control Protocol (TCP)** guarantees reliable delivery.

1
Three-Way Handshake: Initiates connection: **SYN** (synchronize), **SYN-ACK** (synchronize-acknowledge), **ACK** (acknowledge).
2
Sequence Reassembly: TCP reorders packets that arrive out of sequence using sequence numbers.
3
Error Recovery: If a packet is lost in transit, the client requests retransmission of that specific block.
TCP checks sequence numbers to guarantee packets reassemble in order.

Detailed Lecture Notes

The binary system represents the native language of computers, utilizing a base of two numbers. While humans count using ten digits, computers use only zero and one. This mathematical system corresponds directly to the physical states of transistors inside computer microchips. An open circuit allowing current represents a one, while a closed circuit blocking it represents a zero. This clean mechanical mapping makes digital calculations highly reliable and resistant to interference.

Positional notation in binary works similarly to decimal, but each column increases by powers of two. The rightmost column represents ones, the next represents twos, followed by fours, eights, and so on. By adding these place values together, any decimal integer can be represented in binary format. For example, the binary string 101 represents one four and one one, summing to five. This simple base mathematical structure allows computers to perform complex arithmetic using only basic logic gates.

Boolean logic drives all CPU decisions by processing binary inputs to produce a single binary output. Gates like AND, OR, and NOT combine to form complex execution blocks. Millions of these logic gates are etched onto silicon dies to calculate software instructions. This hardware-level math operates at clock speeds measured in billions of cycles per second. Thus, the binary system acts as the bridge connecting physical electricity to mathematical logic.

Think of the TCP handshake as making a standard telephone call connection. First, you say hello to check if the receiver is listening on the line. Second, they reply that they can hear you clearly and ask if you hear them. Third, you acknowledge their reply, and voice conversation starts flowing. If a word gets static and drops, you ask them to repeat that sentence.

04 DNS

DNS: The Web Phonebook

Computers communicate using numeric IP addresses, but humans prefer words. The **Domain Name System (DNS)** acts as the translator.

1
Name Resolution: DNS maps human-readable domain names (like wikipedia.org) to numerical IPs (like 91.198.174.192).
2
DNS Hierarchy: Resolves paths step-by-step: Root servers, TLD (Top-Level Domain) servers (e.g. .org), and Authoritative servers.
3
Caching: Browsers and local systems cache DNS lookups to avoid repeating query steps on every page load.
DNS servers route web inquiries to matching numerical IP directories.

Detailed Lecture Notes

The bit is the fundamental building block of all digital information in computer memory. Short for binary digit, a single bit can only hold a value of zero or one. On their own, individual bits cannot store complex files like pictures or audio. However, by grouping bits together, we can represent larger numbers and characters. This grouping of binary units forms the foundation of modern data storage and transmission.

A byte is a group of exactly eight bits and represents the standard storage unit. With eight binary positions, a single byte can represent two hundred and fifty-six unique states. This capacity is sufficient to encode a single letter, number, or punctuation mark. When you view file sizes, they are measured in multiples of these eight-bit bytes. Kilobytes, megabytes, and gigabytes represent thousands, millions, and billions of these storage blocks.

Computer memory addresses are assigned to individual bytes rather than single bits. This allows the CPU to fetch and write data blocks efficiently during execution loops. If a program needs to store a large number, it utilizes multiple adjacent bytes. This byte-addressable architecture simplifies how memory is managed by the operating system. Knowing how bytes group is essential for managing computer storage capacities and networking budgets.

A simple analogy is looking up a contact name in your smartphone directory. You do not memorize the ten-digit telephone number of every single friend. Instead, you click the name "Mom" and the phone resolves the number. DNS acts as this lookup system, translating "wikipedia.org" into an IP. Without it, you would have to type numbers like `91.198.174.192` to browse.

05 Web Requests

HTTP: Request & Response

Browsers load pages using the **Hypertext Transfer Protocol (HTTP)**, which structures communications between clients and servers.

✓
Request Methods: Browser requests start with methods like **GET** (retrieve a page) or **POST** (submit form details).
✓
Status Codes: Server responses include headers and status codes: **200 OK**, **404 Not Found**, or **500 Server Error**.
✓
Stateless Protocol: Each request-response loop is completely independent. Websites use cookies to store login session records.
Web server processors handle incoming HTTP GET requests continuously.

Detailed Lecture Notes

To represent human language, computers need a consistent system to map numbers to characters. Since hardware only processes binary digits, characters must be encoded as numeric values. A character encoding table acts as a dictionary connecting numbers to specific letters. When you type on a keyboard, a numeric code is sent to the processor. The computer then looks up this numeric code to display the correct symbol on screen.

Without a standardized lookup dictionary, files sent between computers would display as unreadable garbled text. Early computer developers created custom tables, which often caused severe document compatibility problems. These localized encoding systems made international communication difficult and prone to corruption. Therefore, standardizing character maps became a priority for the emerging digital network. Modern systems rely on global standards to guarantee that text renders correctly worldwide.

Character encoding is not limited to letters but also covers numbers and control symbols. Control characters include instructions like carriage returns, tabs, and line breaks. The operating system processes these hidden codes to format text layouts properly on screen. This ensures that formatting remains consistent across different software applications and web browsers. Managing these translation tables is the foundation of global software development and data transmission.

Think of HTTP as ordering food at a drive-through restaurant window. You give the clerk a GET request naming the burger you want. The clerk checks the kitchen server to see if the item is ready. If the item is ready, they hand it over with a 200 OK status code. If they are out of burgers, they reply with a 404 Not Found error.

06 Security

HTTPS & Secure Encryption

Standard HTTP data is sent in plain text, making it vulnerable to interception. **HTTPS** secures the stream using cryptographic protocols.

✦
SSL/TLS Handshake: Establishes a secure connection, verifying server identity using public-key certificates.
✦
Encryption: Scrambles the request-response data packets, preventing local interceptors (e.g. on public Wi-Fi) from reading login details.
✦
Asymmetric Cryptography: Uses public keys to share session keys, shifting to fast symmetric encryption for web transactions.
Cryptographic components handle SSL/TLS decryption on server motherboards.

Detailed Lecture Notes

ASCII stands for the American Standard Code for Information Interchange and was defined in 1963. It was originally designed for telegraph communication but became the standard for early computer systems. ASCII utilizes a seven-bit encoding structure to map characters to numbers. This seven-bit length allows it to represent exactly one hundred and twenty-eight unique characters. These values include the standard English alphabet, digits, punctuation marks, and control codes.

In the ASCII table, uppercase letters are grouped sequentially to make sorting operations easy. For example, uppercase A is mapped to sixty-five, while uppercase B is sixty-six. Lowercase letters follow a similar sequence, starting at decimal ninety-seven with lowercase a. This numeric arrangement allows programmers to manipulate text strings using simple arithmetic. For instance, subtracting thirty-two from a lowercase letter code converts it to uppercase.

Although ASCII is very simple, its small file footprint made it highly efficient for early networks. Because it only required seven bits, it fit comfortably inside standard eight-bit bytes with room to spare. The remaining eighth bit was often used as a parity check to detect transmission errors. This efficiency helped early computers maximize their limited storage and communication bandwidth. Today, ASCII remains the historical baseline upon which modern encodings are built.

A good analogy is sending a secret letter inside a locked steel box. You put the message inside and lock it using a padlocked key. The postal worker can deliver the box but cannot read the note inside. Similarly, HTTPS wraps your browser requests inside an encrypted cryptographic box. Only the host server holds the matching key to unlock and read it.

10 Conclusion

Internet Part 2 Summary

  • **Packets** are the standardized blocks of data routed independently across local and global networks.
  • **TCP** handles reliable packet transfers, executing the 3-way handshake and retransmitting lost packets.
  • **DNS** acts as the address book of the internet, resolving domain names into numerical IP targets.
  • **HTTP** defines the browser client-server request methods (GET/POST) and response codes (200/404).

Detailed Lecture Notes

The primary limitation of ASCII is its tiny seven-bit capacity of one hundred and twenty-eight characters. This small range can only fit the basic English alphabet and common symbols. As computing expanded globally, this English-centric design became a major obstacle. Languages that use accented letters, such as Spanish, French, and German, could not be mapped. This limitation meant ASCII was unable to support the writing systems of most global populations.

Non-Latin scripts like Chinese, Arabic, Cyrillic, and Hindi require thousands of unique characters. Trying to fit these complex writing systems into a 128-character table is mathematically impossible. To solve this, developers created localized eight-bit extensions that mapped characters to the extra values. However, these custom tables clashed with each other and caused documents to display incorrect symbols. A document written in one country would open as scrambled gibberish in another.

Modern text elements, like emojis, also require representation in digital files. Because ASCII was created long before emojis existed, it has no codes allocated for them. Without a larger encoding standard, modern digital expression would be severely limited. These combined shortfalls made it clear that a new universal standard was necessary. Moving beyond ASCII allowed computing to become a truly global, multilingual medium.

A final analogy is to view the entire web as a global postal service. Domain Name Systems act as address directories matching names to physical locations. TCP controls compile packets, ensuring they arrive in sequence without loss. HTTP requests operate as standard order slips between clients and web hosts. Together, these layers ensure the global internet routes data securely and fast.
